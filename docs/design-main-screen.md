# 메인 화면 디자인 — 확정안

> 명세 §7.1이 "초안"으로 남긴 화면 구성을, 검토를 거쳐 확정한 문서.
> 기준: Apple HIG, iOS 기본 날씨앱 1차 레퍼런스 (명세 §7).

## 핵심 모델 — 우산은 아침에 한 번 결정된다

이 앱의 실제 사용 시나리오는 **아침 출근길에 앱을 여는 것**이다. 사용자는 집을 나서며
우산을 챙길지 말지 **한 번** 정하면 된다(퇴근 때 비 와도 아침에 안 챙긴 우산은 생기지 않는다).

따라서 우산 판정의 진짜 로직은:

> **출근 OR 퇴근, 둘 중 하나라도 비 → "우산 챙기세요"**

출근/퇴근을 따로 묻는 게 아니라 **오늘 하루 우산이 필요한가**를 아침에 답한다.
앱 이름 "우산챙겨?"의 답이 화면 맨 위에 **하나의 결론**으로 나온다.

## 레이아웃 (정보 위계)

```
┌─────────────────────────┐
│ 우산챙겨?            ⚙︎ │
│                         │
│        ☔️               │   ← 오늘의 결론 아이콘
│   우산 챙기세요          │   ← 결론 (출근 OR 퇴근 비). 가장 큼.
│   퇴근길에 비가 와요      │   ← 왜인지 (어느 시점 비인지 정직하게)
│                         │
│  ┌────────┐ ┌────────┐ │
│  │출근 8:30│ │퇴근19:00│ │   ← 근거 두 시점 (보조, 작게)
│  │🌤 21° 0%│ │🌧 24°70%│ │      탭하면 시간별 흐름 펼침(§7.1)
│  └────────┘ └────────┘ │
└─────────────────────────┘
```

1. **상단 결론** — 가장 큰 시각적 무게. 출근/퇴근 둘 중 하나라도 비면 "우산 챙기세요",
   둘 다 아니면 "우산 필요 없어요". **명령/조언형 톤** (명세 §5 푸시 문구와 일관).
2. **결론 부연 한 줄** — "퇴근길에 비가 와요" 처럼 *어느 시점*이 비인지 정직하게.
   둘 다 비면 "오늘 종일 비 소식", 둘 다 맑으면 생략하거나 "오늘은 맑아요".
3. **출근/퇴근 두 카드** — 동등하게 작게. 각 시점의 날씨·기온·강수확률. 근거 역할.
   - 탭 시 점진적 공개: 해당 시각 전후 시간별 흐름 (출근 2h전~1h후 / 퇴근 1h전~2h후).

## 톤 & 스타일 (명세 §7)
- Apple HIG. SF Pro(시스템 폰트), 표준 여백, 최소 색상.
- 절제와 위계: 결론 하나에 무게를 몰아주고 나머지는 작게.
- 결론 문구 톤: **명령/조언형** — "우산 챙기세요" / "우산 필요 없어요".

## 구현 메모
- 서버 `/forecast`는 이미 `morning.needUmbrella` / `evening.needUmbrella`를 각각 반환.
  앱은 `morning.needUmbrella || evening.needUmbrella`로 상단 결론을 계산. **추가 API 불필요.**
- 부연 문구는 어느 카드의 needUmbrella가 true인지로 결정.
- 데이터 없는 슬롯(null) 처리: 그 카드는 "불러오는 중" 또는 정보 부분 표시.
